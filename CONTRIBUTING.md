# Contributing

Thank you for your interest in contributing to AI-PROTOCOLS.

This repository is experimental. Contributions should preserve the main objective: compact, verifiable documentation protocols with no runtime dependency.

## How to Contribute

- Open an issue before sending a PR that changes protocol behavior.
- Keep changes small and focused.
- Update examples, manual tests, or benchmarks when a change affects observable behavior.
- Record relevant changes in CHANGELOG.md.
- Preserve compatibility within the same protocol version.

## Accepted Scope

- protocol specifications.
- compact examples.
- manual tests.
- manual benchmarks.
- governance documents.
- clarity improvements that do not change semantics.

## Out of Scope

- runtime implementation.
- bridge or transport.
- product code.
- long prompts or prompt libraries.
- definitive model rankings.

## Style

- Prefer direct language.
- Use stable fields and short lists.
- Declare limits and non-goals when relevant.
- Do not introduce new semantics in editorial changes.
